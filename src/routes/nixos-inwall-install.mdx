> 本文主要介绍一些在受限网络中安装和使用NixOS的实用技巧。面向初次安装NixOS的有一定Linux基础的用户或外宾NixOS用户。

# 名词解释

+ Channel

  狭义指 [nix channels](https://nixos.wiki/wiki/Nix_channels)，属于旧的 nix- 系列命令。引申为 nixpkgs 的所有branch, 包括 `master`, `nixos-unstable` 等。在某些镜像站的文档中，可能与二进制缓存（见下）混淆。如 [ustc镜像站](https://mirrors.ustc.edu.cn/help/nix-channels.html)，使用不同path区分channel和二进制缓存（如 /nixos-unstable 和 /store）。

+ Binary Cache (二进制缓存)

  储存预先编译的二进制的服务。如果无法正常访问，但 Channel 能够正常访问，nix就会开始在本地进行大量编译，这通常不是预期的。

幸运的是，在部分国家或地区，这两者**都**无法保证正常访问。

# 正文

分为两个解决途径，翻墙 和 镜像。前者是一劳永逸之道，所以先说，后者通常会面临缓存镜像时不时同步滞后而无法命中的问题，同样会造成本地大量重新构建。

## 翻

### 初次安装

我无法翻墙，如何能安装翻墙软件？鸡蛋问题困扰着许多人。幸运的是，你可以选择以下方法：

+ 安卓VPN共享

  部分安卓手机可以直接通过USB或者热点分享本机的VPN代理。或者获取root权限的手机通过应用 `VPN hotspot` 来进行配置。对于安装场景，临时代理热点通常是可以接受的。

+ 从国内镜像站抠出代理软件

  如果你有现成的翻墙软件配置，建议选择此方法。代理软件多数使用golang，具备静态链接的天然优势，所以假定这个方案可以作为墙内获取翻墙软件的通用方案。

```bash
# 或者 pkgs.sing-box / pkgs.mihomo, 都可以
nix-build -E '{pkgs ? import <nixpkgs> {}}: pkgs.dae' --option substituters "https://mirrors.ustc.edu.cn/nix-channels/store"
```

然后在 `result` 文件夹中取用。

值得一提的是Nix Daemon 一般不吃环境变量，如果你不开透明代理(如TUN或者基于eBPF或者tproxy的代理)，那需要额外的配置使nix走代理：

你可以参考 [issue](https://github.com/NixOS/nixpkgs/issues/27535#issuecomment-1178444327)，

```bash
sudo mkdir /run/systemd/system/nix-daemon.service.d/
cat << EOF >/run/systemd/system/nix-daemon.service.d/override.conf  
[Service]
Environment="http_proxy=socks5h://localhost:7891" # 假设是你的socks代理端口
Environment="https_proxy=socks5h://localhost:7891"
Environment="all_proxy=socks5h://localhost:7891"
EOF
```

或者使用 `sudo systemctl edit --runtime nix-daemon` 进行同样更改。

最后应用更改使nix吃代理环境变量，就可以正常执行安装步骤，无需考虑网络环境问题。

```bash
sudo systemctl daemon-reload
sudo systemctl restart nix-daemon
```

### 使用时

nixpkgs已经包含大量相关工具和设施，相见[tracking issue](https://github.com/NixOS/nixpkgs/issues/287770)。使用 [nix search](https://search.nixos.org/options?) 搜索相关选项并开启服务。如果你有能力，请尽量使用无图形界面的工具，因为GUI工具相比其实际生效的功能部件并没有提高更多效率，并且容易出现问题（仅在nixos的语境下，如 clash-verge 的GUI选项并不是完全生效）

如果你已经对相关工具有了解，请自行选择，如果没有，以下可以考虑：

+ dae: eBPF 透明代理。有完善的社区支持，见[flake](https://github.com/daeuniverse/flake.nix)

+ sing-box: 有nixos模块，支持大部分协议，支持透明代理

+ mihomo (clash-meta): 广泛兼容的订阅格式。和singbox坐一桌

## 镜像

+ [flake](https://wiki.nixos.org/wiki/Flakes) 是一种狭义 nix channel 的替代。作用是更好地锁定版本。

<Emph type="note">如果你使用flake，则无法正常使用channel镜像，但是由于它支持所有git仓库，你可以指定任意国内的nixpkgs git仓库镜像。但是我没找到。</Emph>

### 初次安装

出于上述原因，我们先假定你不使用flake了。

参考 [ustc镜像站](https://mirrors.ustc.edu.cn/help/nix-channels.html) 的添加流程。你也可以选择国内的 sjtu、tuna、bfsu 的镜像，看地区和个人喜好来。

```
sudo -i
nix-channel --add https://mirrors.ustc.edu.cn/nix-channels/nixos-unstable nixos # nixos 是安装默认使用的channel
nix-channel --update  # 更新并解包频道
```

然后继续进行安装步骤。在 `nixos-install` 的步骤，请添加额外的命令行参数：

```bash
--option extra-substituters "https://mirror.sjtu.edu.cn/nix-channels/store"
```

<Emph type="note">有些教程中可能会使用 `substituters` 而不是 `extra-*`，前者会删除nix official自带的二进制缓存，在只有国内镜像缓存且不命中的情况下会执行大量本地编译。</Emph>

### 使用时

进入系统后，需要重复上述安装时的步骤。确保你创建的用户没有 `nix channel --list` 输出，以防止和root的channel混淆。
